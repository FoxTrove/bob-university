## Strategy Overview

The launch follows a **two-campaign approach** with a **soft launch phase** for risk mitigation:

| Campaign | Goal | Target Audience |
|----------|------|-----------------|
| **Soft Launch** | Test with real users, fix issues | 50-100 beta testers |
| **App Launch** | Drive downloads, re-engage lapsed users | Existing email list, previous subscribers |
| **Lead Generation (Webinar)** | Build new email list, convert to subscribers | New stylists via social media |

The webinar ("Grand Slam Offer") is designed as a **high-trust, high-value free offer** that solves a surface-level problem (content creation anxiety) to expose the deeper problem (skill gaps) that the app solves.

### Core Strategy Pillars

1. **Expert-Led Positioning** ‚Äî Ray as professional authority, not just influencer
2. **SMS + Email Marketing** ‚Äî Dual-channel for 8x better engagement
3. **Soft Launch De-risking** ‚Äî Two beta rounds before public launch
4. **Optimized Webinar Funnel** ‚Äî Research-backed conversion targets
5. **Micro-Influencer Partnerships** ‚Äî Authentic voices in the stylist community
6. **Referral Program** (Post-Launch) ‚Äî Viral growth loop for Month 2+

## Referral Program (Post-Launch: March)

A dual-sided referral program creates a viral growth loop and reduces CAC.

### Structure

| Who | Reward |
|-----|--------|
| **Referrer** (existing user) | 1 free month of premium |
| **Referee** (new user) | 20% off first month |

### User Flow

1. User completes a module or earns badge ‚Üí prompt: "Share your achievement?"
2. User taps "Share" ‚Üí generates unique referral link
3. Friend clicks link ‚Üí downloads app ‚Üí signs up
4. Friend subscribes ‚Üí both get rewards automatically
5. Referrer notified: "You earned a free month!"

### Implementation Requirements

**Database** (already in IMPLEMENTATION_PLAN.md):
```sql
CREATE TABLE referral_codes (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES auth.users(id),
  code TEXT UNIQUE NOT NULL,
  uses INTEGER DEFAULT 0,
  reward_months INTEGER DEFAULT 1,
  referee_discount_percent INTEGER DEFAULT 20,
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE referral_redemptions (
  id UUID PRIMARY KEY,
  code_id UUID REFERENCES referral_codes(id),
  referee_id UUID REFERENCES auth.users(id),
  referrer_rewarded BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT now()
);
```

**App Changes**:
- Share button on achievement/completion screens
- Referral code input during signup
- "My Referrals" section in profile

**Tracking**:
- Viral coefficient (K-factor): aim for >0.3
- Referrals per user
- Conversion rate of referred users
